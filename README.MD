# 🧪 API Test Automation Project

Automated API testing project using **Jest** and **Supertest**, focused on validating REST APIs with functional, contract, and negative testing strategies.

This project demonstrates real-world API testing practices, CI/CD pipeline integration, and test reporting via GitHub Actions.

---

## 🚀 Tech Stack

- Node.js
- Jest
- Supertest
- Faker-based test data factory
- JSON Schema validation
- GitHub Actions (CI/CD)
- HTML test reporting (`jest-html-reporters`)

---

## 📌 Project Goal

The goal of this project is to simulate a professional API test automation framework with:

- Functional tests (happy paths)
- Negative tests (invalid inputs and edge cases)
- Contract/schema validation
- CI execution via GitHub Actions
- HTML reporting artifacts

---

## 📁 Project Structure

# 📁 Project Structure

This project follows a layered architecture to separate responsibilities between tests, services, utilities, and configuration.

---

## 🧪 Test Layer

### `specs/`

Contains all test suites responsible for validating API behavior.

specs/
└── products.spec.js # Main test suite (Products API)


- Includes:
  - Functional tests
  - Negative tests
  - Contract/schema tests
  - Data-driven scenarios

---

## 🔌 Service Layer

### `services/`

Responsible for API communication abstraction.

services/
└── productService.js # API abstraction layer


- Encapsulates HTTP requests
- Centralizes API endpoints
- Improves reuse and maintainability

---

## 🛠 Utilities Layer

### `utils/`

Helper functions and reusable validation logic.

utils/
├── schemaValidator.js # JSON schema validation helper
└── validators/
└── product.validator.js # Response validation rules


- Validates API responses
- Ensures data consistency
- Supports contract testing

---

## 📦 Schemas Layer

### `schemas/`

JSON Schema definitions used for contract validation.

schemas/
├── product.schema.json # Single product schema
└── products.schema.json # Products list schema


- Ensures API response structure
- Prevents breaking contract changes
- Used in contract tests

---

## 🏭 Factories Layer

### `factories/`

Responsible for generating test data dynamically.

factories/
└── product.faker.js # Fake product generator


- Creates valid and invalid payloads
- Supports negative testing scenarios
- Improves test scalability

---

## ⚙️ Configuration Layer

### `config/`

Centralized configuration files.

config/
└── base.config.js # Base API URL configuration


- Stores environment settings
- Defines API base URLs
- Supports multi-environment setup

---

## 📊 Reports Layer

### `reports/`

Generated test execution reports.

reports/
└── index.html # HTML report (generated by Jest)


- Auto-generated after test execution
- Contains test results summary
- Used in CI/CD pipeline artifacts

## 🔗 API Under Test

This project uses the public API:

- https://fakestoreapi.com

### ⚠️ Important Limitation

The API is protected by Cloudflare security rules.

As a result:
- Requests may work locally
- GitHub Actions runners may receive `403 Forbidden`
- This is caused by external anti-bot protection, not the test framework

This behavior was confirmed using `curl` inside CI pipelines.

---

## 🧪 Test Types

### ✅ Functional Tests
Validate expected API behavior:

- GET /products returns a valid list
- GET /products/:id returns correct product
- POST /products creates a product successfully

---

### ❌ Negative Tests
Validate system behavior with invalid inputs:

- Invalid product IDs
- Invalid payloads
- Non-existent endpoints

> Some negative cases are marked as known API limitations due to external API behavior.

---

### 📄 Contract Tests
Validate API response structure using JSON Schema:

- Ensures response consistency
- Validates required fields and types
- Prevents breaking changes in API contract

## ⚙️ CI/CD (GitHub Actions)

This project includes a CI pipeline that:

- Installs dependencies
- Executes Jest tests
- Generates HTML report
- Uploads artifacts
- Deploys report to GitHub Pages

### Workflow triggers:

- Push to master
- Pull requests to master

## ⚠️ Known Issue (CI)

GitHub Actions may fail when calling FakeStoreAPI:

HTTP 403 - Cloudflare challenge

### Root cause:

External API blocks non-browser requests from CI environments.

#### Impact:

- Tests may pass locally
- Fail in CI environment
- Not related to test implementation
